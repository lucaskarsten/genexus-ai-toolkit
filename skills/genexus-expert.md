## USER CONTROLS (UC) — COMPLETE GUIDE

### .view File Structure

A UC has **two parts in the same file**:

```xml
<!-- PART 1: Definition (XML) — properties, events, AfterShow script -->
<Definition auto="false">
    <Property Name="ucid"       Type="string" Default="" />
    <Property Name="Label"      Type="string" Default="" />
    <Property Name="DataJson"   Type="string" Default="" />

    <Event Name="OnClick" />

    <Script Name="AfterShow" When="AfterShow">
        (function () {
            var ucid = this.ucid;
            var self = this;
            var el   = document.querySelector('[data-ucid="' + ucid + '"]');
            if (!el) return;
            if (el.getAttribute('data-uc-init') === '1') return;
            el.setAttribute('data-uc-init', '1');

            /* ... logic here ... */

        }).call(this);
    </Script>
</Definition>

<!-- PART 2: Screen Template (HTML) — never <script> here -->
<div class="my-wrapper" data-ucid="{{ucid}}">
    <span class="my-label">{{Label}}</span>
    <!-- long data in invisible div, never in attribute -->
    <div id="my-data-{{ucid}}" style="display:none">{{DataJson}}</div>
</div>
```

### The 5 Fundamental Laws

**Law 1 — `<script>` in Screen Template NEVER executes.**
HTML5 ignores scripts injected via innerHTML. All JS goes exclusively in `<Script When="AfterShow">`.

```xml
<!-- ❌ WRONG: never works -->
<div>...</div>
<script>alert('never runs');</script>

<!-- ✅ CORRECT -->
<Definition auto="false">
    <Script Name="Init" When="AfterShow">
        (function() { /* logic here */ }).call(this);
    </Script>
</Definition>
<div>...</div>
```

**Law 2 — `AfterShow` re-executes on EVERY postback.**
Without a re-init guard, event listeners multiply on each postback.

```javascript
var el = document.querySelector('[data-ucid="' + ucid + '"]');
if (!el) return;
if (el.getAttribute('data-uc-init') === '1') return;  // ← CRITICAL
el.setAttribute('data-uc-init', '1');
```

**Law 3 — `ucid` is mandatory.**
Without it, IDs become `btn-` (no suffix), two instances on the same page collide.

In WebPanel, always set ucid first:
```genexus
UCMyControl.ucid  = !'UniqueId'    // FIRST
UCMyControl.Label = &MyVariable
```

**Law 4 — Long text goes in CONTENT, never in HTML attribute.**
Quotes, `\n`, `<`, `&` inside attributes break HTML.

```xml
<!-- ❌ WRONG -->
<div data-text="{{LongText}}">...</div>

<!-- ✅ CORRECT -->
<div id="my-data-{{ucid}}" style="display:none">{{LongText}}</div>
```

**Law 5 — Backslash in regex disappears in GeneXus.**
```javascript
/* ✅ Always safe */
var re = new RegExp(String.fromCharCode(10), 'g');  // \n
var re = new RegExp(String.fromCharCode(9),  'g');  // \t
```

### Properties — Types and Conventions

| GX Type | Use | Example |
|---------|-----|---------|
| `string` | text, JSON, boolean | `<Property Name="Label" Type="string" Default="" />` |
| `string` (boolean) | compare with `=== "true"` | `<Property Name="IsActive" Type="string" Default="false" />` |
| JSON/Collection | `&SDT.ToJson()` in GX, `JSON.parse()` in JS | `<Property Name="Items" Type="string" Default="[]" />` |
| Number | `Str(&N, 20, 2)` in GX | `<Property Name="Value" Type="string" Default="0" />` |

### Complete IIFE Template

```javascript
(function () {
    /* ── setup ──────────────────────────── */
    var ucid = this.ucid;
    var self = this;

    var el = document.querySelector('[data-ucid="' + ucid + '"]');
    if (!el) return;
    if (el.getAttribute('data-uc-init') === '1') return;
    el.setAttribute('data-uc-init', '1');

    /* ── utilities ───────────────────────── */
    function esc(s) {
        return String(s == null ? '' : s)
            .replace(/&/g, '&amp;').replace(/</g, '&lt;')
            .replace(/>/g, '&gt;').replace(/"/g, '&quot;');
    }

    function decodeHtml(s) {
        var d = document.createElement('div');
        d.innerHTML = s;
        return d.textContent || d.innerText || '';
    }

    function norm(s) {  /* normalize accents for search */
        return String(s || '').normalize('NFD')
            .replace(new RegExp('[' + String.fromCharCode(768) + '-' + String.fromCharCode(879) + ']', 'g'), '')
            .toLowerCase();
    }

    /* ── parse data ──────────────────────── */
    var items = [];
    try {
        var raw = (this.Items || '').trim();
        if (raw) items = JSON.parse(raw);
    } catch (e) { items = []; }

    /* ── main logic ──────────────────────── */

    /* ── dispatch GeneXus event ──────────── */
    function fireSelect(id) {
        var ep = document.querySelector('input[name="vEVENTPARAM"]');
        if (ep) ep.value = id;
        if (self && typeof self.OnSelect === 'function') self.OnSelect();
    }

    /* ── close on outside click ──────────── */
    document.addEventListener('click', function (e) {
        if (!el.contains(e.target)) close();
    });

}).call(this);
```

## WEB PANEL / WEB COMPONENT — `.events.gx`

### Standard Structure

```genexus
Event Start
    Do 'BuildControls'
EndEvent

Event Refresh
    Do 'BuildControls'
EndEvent

Sub 'BuildControls'
    Do 'BuildMyControl'
EndSub

Sub 'BuildMyControl'
    // 1. ucid ALWAYS first
    UCMyControl.ucid  = !'my-control-id'

    // 2. simple properties
    UCMyControl.Label = &MyVariable
    UCMyControl.IsActive = Iif(&Flag = 1, !'true', !'false')

    // 3. collections — NEVER manual concatenation
    &JsonItems = &MySDT.ToJson()
    UCMyControl.Items = &JsonItems
EndSub

Event UCMyControl.OnSelect
    &EventId = &EventParam
    Do Case
        Case &EventId = !'action-a'
            Do 'HandleActionA'
        Case &EventId = !'action-b'
            Do 'HandleActionB'
    EndCase
EndEvent
```

### SDT Collection → UC

```genexus
/* ✅ CORRECT: SDT.ToJson() */
For Each MyTable
    &SdtItem.Id    = MyTable.Id
    &SdtItem.Label = MyTable.Name
    &SdtCollection.Add(&SdtItem)
EndFor
UCMyControl.Items = &SdtCollection.ToJson()

/* ❌ WRONG: manual concatenation */
UCMyControl.Items = !'[{"id":"' + &Id + '","label":"' + &Name + '"}]'
/* → breaks when &Name has quotes, accents, etc. */
```

### Web Panel Naming

| Prefix | Type | Example |
|--------|------|---------|
| `mpg` | Master Page | `mpgYourApp_V2` |
| `wbc` | Reusable Web Component | `WbcNavHeader`, `WbcNavMenu` |
| `wbp` | Specific Web Panel | `WbpDashboard`, `WbpReports` |

### Events lifecycle

```genexus
Event Start
    Do 'BuildControls'
EndEvent

Event Refresh
    Do 'BuildControls'
EndEvent

// User-defined events (e.g. grid row selection, button click)
Event GridMyGrid.Click
    &SelectedId = GridMyGrid.MyIdAttribute
    Do 'HandleSelection'
EndEvent

Sub 'BuildControls'
    // Always: ucid FIRST, then other properties
    UCMyControl.ucid  = !'ctrl-id'
    UCMyControl.Items = &MySDT.ToJson()
EndSub
```

**Postback ordering**: `Start` fires once on load. `Refresh` fires on every postback. Both should delegate to the same `BuildControls` sub to avoid duplication.

### Error handling

```genexus
Sub 'HandleAction'
    If &Result.HasErrors()
        msg(&Result.Errors.Item(1).Description)
        Return
    EndIf
    // success path
    Refresh
EndSub
```

Use `msg()` for user-visible messages. Use `Return` to stop execution early. Avoid duplicating error-checking logic across events — extract it into a shared sub.

### WBP ↔ UC pub/sub (example with UcNavSearch)

```genexus
// WebPanel with a UcNavSearch that filters a grid

Event Start
    Do 'BuildSearch'
    Do 'BuildGrid'
EndEvent

Event Refresh
    Do 'BuildSearch'
    Do 'BuildGrid'
EndEvent

Sub 'BuildSearch'
    UCSearch1.ucid        = !'search-nav'
    UCSearch1.Placeholder = 'Search items...'
    UCSearch1.Items       = &SearchItems.ToJson()
EndSub

Sub 'BuildGrid'
    For Each Item
        Where Item.Name Like '%' + &FilterTerm + '%'
        &GridItem.Id   = Item.Id
        &GridItem.Name = Item.Name
        &GridItems.Add(&GridItem)
    EndFor
EndSub

Event UCSearch1.OnSelect
    &FilterTerm = &EventParam      // value passed via vEVENTPARAM
    Refresh
EndEvent
```

`&EventParam` receives whatever the UC put in `vEVENTPARAM` (for UcNavSearch, the `id` of the selected item). The `Refresh` re-runs `BuildGrid` with the new filter.

### Output

Generated Web Panel source lives in `output/WBP/<WbpName>_<description>.view`. The `.events.gx` content (event subs and logic) goes in `output/WBP/`.

### Events Lifecycle

Event execution order on first page load: `Start` → `Load` → user events (button clicks, grid row clicks, UC events). On postback triggered by a `Refresh` call: `Refresh` → `Load`.

- **`Start`** fires once per full page load. Use it to initialize variables, build UC controls, and set default filter values. It does not fire again on postback.
- **`Load`** fires on grid load and on every `Refresh`. Bind grid data inside a `For Each` block here (or inside a sub called from `Load`).
- **`Refresh`** fires on every postback. Delegate to the same `BuildControls` sub used in `Start` to avoid duplication.
- **`BuildControls`** (sub) re-runs on postback when the grid needs re-binding. Call it from both `Start` and `Refresh` so UC properties are re-applied after each round-trip.

```genexus
Event Start
    Do 'BuildControls'
    &FilterTerm = ''          // initialize default filter
EndEvent

Event Load
    // grid data binding runs here on every load/refresh
    For Each Item
        Where Item.Name Like '%' + &FilterTerm + '%'
        // bind grid attributes
    EndFor
EndEvent

Event Refresh
    Do 'BuildControls'
EndEvent

Sub 'BuildControls'
    UCMyControl.ucid  = !'my-ctrl-id'
    UCMyControl.Items = &MySDT.ToJson()
EndSub
```

### Error Handling

Use `msg()` inside event subs to display user-facing errors. Use an early-return guard to stop execution before reaching grid operations or DB writes when input is invalid.

```genexus
Event 'Search'
  if SearchTerm.IsEmpty()
    msg("Informe um termo de busca.")
    return
  endif
  // proceed
EndEvent
```

Check error conditions before grid operations:

```genexus
Sub 'HandleAction'
    If &Result.HasErrors()
        msg(&Result.Errors.Item(1).Description)
        Return
    EndIf
    Refresh
EndSub
```

Pattern: validate inputs at the top of each event, return early on failure, keep the success path linear below the guards. Avoid duplicating error checks — extract shared validation into a sub.

### Pub/Sub with UcNavSearch

UcNavSearch publishes `OnSelect` when the user picks a search result. The WBP subscribes by naming its event after the UC instance: `UCNavSearch1.OnSelect`.

Wiring pattern — the WBP receives the search event and reloads the grid with the new filter:

```genexus
Event 'UCNavSearch1.OnSelect'
  SearchGrid.CurrentPage = 1
  SearchGrid.Load()
EndEvent
```

The WBP reads `UCNavSearch1.ItemSelected` (set by the UC before firing `OnSelect`) to get the selected value, or reads `&EventParam` for the passed ID. This value is stored in a filter variable (e.g., `&FilterTerm`) that the grid's `For Each` uses in its `Where` clause.

Full wiring example:

```genexus
Event Start
    Do 'BuildSearch'
    &FilterTerm = ''
EndEvent

Event Refresh
    Do 'BuildSearch'
EndEvent

Sub 'BuildSearch'
    UCNavSearch1.ucid        = !'nav-search'
    UCNavSearch1.Placeholder = 'Search items...'
    UCNavSearch1.Items       = &SearchItems.ToJson()
EndSub

Event 'UCNavSearch1.OnSelect'
    &FilterTerm = UCNavSearch1.ItemSelected
    SearchGrid.CurrentPage = 1
    SearchGrid.Load()
EndEvent

Event Load
    For Each Item
        Where Item.Name Like '%' + &FilterTerm + '%'
        // bind grid row
    EndFor
EndEvent
```

The UC publishes `OnSelect`; the WBP subscribes via the `<UcInstanceName>.OnSelect` event name pattern. No explicit subscribe call is needed — GeneXus wires these by naming convention.

### Output / File Path

Web Panel event code lives in `.events.gx` files in GeneXus 18 format. In this toolkit, generated output goes to:

```
output/WBP/<PanelName>.events.gx
```

Example: `output/WBP/WbpSearchWithNavSearch.events.gx`

The compiled result in GeneXus lives under the KB's generated web folder (path set by `GX_COMPILER_OUTPUT`). The `.events.gx` file is the source-of-truth artifact to submit for KB import or review.

## JAVASCRIPT ES5 — ADVANCED PATTERNS

### Mandatory Syntax Rules

```javascript
/* ✅ ES5 correct */
var name = 'value';
var fn = function(a, b) { return a + b; };
var arr = [1, 2, 3];
var obj = { key: 'value' };

/* ❌ Prohibited in GeneXus */
let x = 1;                              /* let */
const Y = 2;                            /* const */
var fn = (a) => a * 2;                  /* arrow function */
var s = `template ${variable}`;         /* template literal */
var { a, b } = obj;                     /* destructuring */
var newArr = [...arr];                  /* spread */
class MyClass { }                       /* class syntax */
```

### HTML Escape (always for innerHTML)

```javascript
function esc(s) {
    return String(s == null ? '' : s)
        .replace(/&/g, '&amp;')
        .replace(/</g, '&lt;')
        .replace(/>/g, '&gt;')
        .replace(/"/g, '&quot;');
}
panel.innerHTML = '<span class="label">' + esc(item.label) + '</span>';
```

### Decode HTML Entities

```javascript
function decodeHtml(s) {
    var d = document.createElement('div');
    d.innerHTML = s;
    return d.textContent || d.innerText || '';
}
var realText = decodeHtml(dataDiv.innerHTML);
```

### Regex Without Backslash

```javascript
var newline  = String.fromCharCode(10);   /* \n */
var tab      = String.fromCharCode(9);    /* \t */
var reNewline = new RegExp(String.fromCharCode(10), 'g');
text = text.replace(reNewline, '<br>');
```

### Accent Normalization for Search

```javascript
function norm(s) {
    var base = String(s || '').normalize('NFD');
    return base.replace(
        new RegExp('[' + String.fromCharCode(768) + '-' + String.fromCharCode(879) + ']', 'g'), ''

```

### Tooltip (tippy.js with fallback)

```javascript
function applyTooltip(el, text) {
    if (window.tippy) {
        tippy(el, { content: text, placement: 'right', theme: 'light' });
    } else {
        el.title = text;
    }
}
```

### Inject CSS via JS (animations in UC)

```javascript
function injectCss(id, css) {
    if (document.getElementById(id)) return;  /* guard */
    var style = document.createElement('style');
    style.id = id;
    style.textContent = css;

}
```

- [ ] Unique CSS prefix (3-4 chars) on all UC classes

- [ ] BEM kebab-case on all classes — no PascalCase, no camelCase
- [ ] No nested BEM elements (`__a__b` prohibited)
- [ ] Module prefix on module-specific classes

### Before delivering any WBP/WBC

- [ ] `ucid` is the first property set in each assembly sub

## GeneXus Language Quick Reference (from nexa)

> Digest of the most-used constructs. For the full spec, load `skills/nexa/nexa/` or consult the reference files listed below.

### Attribute type markers (Transactions)

| Marker | Meaning | Example |
|---|---|---|
| `Name*` | Primary Key — unique, not null, must come first | `CustomerId*` |
| `Name!` | Description attribute — displayed in prompts and lists | `CustomerName!` |
| `Name?` | Nullable — allows null; forbidden on PK | `CustomerPhone?` |
| `Name []` | Foreign Key — infers type from referenced transaction | `CountryId []` |
| `Formula = '...'` | Computed attribute — not stored unless `Redundant = 'True'` | `TotalAmount` |

Full reference: `skills/nexa/nexa/references/common-attribute-types.md`

### Core rules (Transactions and Procedures)

```genexus
Parm(in: &CustomerId, out: &CustomerName);     // object signature
CustomerName = !'Active' IF CustomerStatus = 1; // Assign
Default(InvoiceDate, &Today);                   // Default value
Error('Price must be positive') IF ProductPrice <= 0;  // Validation
Msg('Discount applied') IF DiscountPercentage > 0;     // Info message
NoAccept(CustomerPassword) IF &IsGuest;         // Read-only field
Serial(InvoiceItemId, InvoiceLastLine, 1);      // Sequential numbering
Add(TotalAmount, ItemAmount);                   // Accumulate
Subtract(StockQuantity, OrderQuantity);         // Decrement
Error_Handler('HandleError');                   // Proc error handler
```

Full reference: `skills/nexa/nexa/references/common-rules.md`

### Rule triggers (Transaction lifecycle)

| Trigger | When |
|---|---|
| `BeforeValidate` | Before the validation process starts |
| `BeforeInsert` / `BeforeUpdate` / `BeforeDelete` | Before each CRUD operation |
| `AfterValidate` | After user confirmation, before CRUD |
| `AfterInsert` / `AfterUpdate` / `AfterDelete` | Immediately after each CRUD |
| `AfterComplete` | After the Logical Unit of Work commits |
| `AfterLevel [level <attribute>]` | After completing a sub-level |

```genexus
UserRegistrationDate = &Today ON BeforeInsert;
Msg(Format('Saved: %1', CustomerId)) ON AfterComplete;
```

### Common commands

```genexus
// For Each
For Each Customer
    Where Customer.Active = 1
    &Count += 1

// If
If &Value > 0
    Do 'ProcessPositive'
Else
    Do 'ProcessNegative'
EndIf

// Do Case
Do Case
    Case &Status = 'A'
        Do 'HandleActive'
    Case &Status = 'I'
        Do 'HandleInactive'
    Otherwise
        Do 'HandleUnknown'
EndCase

PrcMyProcedure(&Param1, &Param2)
&Result = PrcMyFunction(&Param)      // function-style (last parm is out:)
```
