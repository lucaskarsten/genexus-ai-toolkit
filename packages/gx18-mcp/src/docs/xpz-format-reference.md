# XPZ Format Reference

Complete reference for the GeneXus 18 XPZ archive format — XML schema, Part type GUIDs, per-object
structure, variable typing, and silent-import failure modes.

**See also:** `gx18://docs/xpz-workflow` for the `gx_export → gx_read_xpz → gx_patch_xpz → gx_import` round-trip guide.

**Reference file:** `C:\KBs\FoccoLojas_02\FoccoLojas_03.xpz` — 461 real objects (Domains, SDTs,
Procedures, UCs, WBCs, DSOs, External Objects) extracted from the active FoccoLojas_03 KB.

---

## 1. What is an XPZ

- A **ZIP file** containing a single `.xml` (named after the archive)
- UTF-8 with BOM + CRLF line endings
- Generated by the GX18 IDE (Knowledge Manager → Export) or via `gx_export`

> **Warning:** `Expand-Archive` (PowerShell) refuses `.xpz` — always use `[System.IO.Compression.ZipFile]`.

---

## 2. XML Root (`<ExportFile>`)

```xml
<?xml version="1.0" encoding="utf-8"?>
<ExportFile>
  <KMW>
    <MajorVersion>4</MajorVersion>
    <MinorVersion>0</MinorVersion>
    <Build>177934</Build>
  </KMW>
  <Source kb="9ced92e5-f7b4-47c3-9af8-c84b8f5c565a"
          username="COMPUSOFT\lucas.karsten"
          UNCPath="\\host\C$\KBs\FoccoLojas_03">
    <Version guid="64e77778-2ac3-4a8b-b0c3-e641b98d1943" name="FoccoLojas_03" />
  </Source>
  <Objects>
    <!-- 1..N <Object> elements -->
  </Objects>
</ExportFile>
```

> **PowerShell path:** `$xml.ExportFile.Objects.Object` — NOT `$xml.KMW.Objects.Object`.

When generating a minimal XPZ for import, include only the objects you intend to import in `<Objects>`.
Re-including unrelated host objects re-versions them unnecessarily.

## 3. `<Object>` Attributes

| Attribute | Required | Rule |
|---|---|---|
| `guid` | ✅ | **Generate a new GUID when crossing KBs** — collision silently overwrites a different object |
| `name` | ✅ | Object name (e.g. `SdtVenMeusProjetosEmNegociacao`) |
| `type` | ✅ | EntityType GUID (see table in §4) |
| `description` | ✅ | Free-text description |
| `parent` | ✅ | Parent module/folder name (e.g. `VEN`) |
| `parentType` | ✅ | Parent type GUID (folder = `00000000-0000-0000-0000-000000000008`) |
| `parentGuid` | ✅ | GUID of the parent module/folder |
| `moduleGuid` | — | Same as `parentGuid` for module-level objects |
| `fullyQualifiedName` | — | Namespace-qualified name (e.g. `VEN.SdtVenMeusProjetosEmNegociacao`) |
| `user` | — | `DOMAIN\user` — author stamp |
| `versionDate` | — | Current version date (ISO 8601 UTC) |
| `lastUpdate` | ✅ | Must be **later than** the existing version in the target KB for import to take effect |
| `checksum` | ✅ | MD5 of content — GX does **not** validate cryptographically; zeroing with `"00000000000000000000000000000000"` works |

## 6. Per-Object Structure

### UserControl (`type` = `562f4793-...`)

Two Parts:

```xml
<Object name="UCMenuUsuario" type="562f4793-aabe-449f-8821-fc77e550698e"
        guid="..." description="..." parent="UserControls" ...>

  <!-- Part 1: HTML + CSS template -->
  <Part type="3dd92fe7-b095-44d3-9fa0-8488fa3f0c67">
    <Source><![CDATA[
      <style>
        .profile-menu__item { color: var(--colors_text-primary); }
      </style>
      <div class="app-header__user-menu" data-ucid="{{ucid}}">
        <!-- HTML -->
      </div>
    ]]></Source>
  </Part>

  <!-- Part 2: Property definitions + AfterShow / Methods scripts -->
  <Part type="8e9e4a7c-a4d3-4c36-8e8e-fb6702402f63">
    <Source><![CDATA[<Definition auto="false">
      <Property Name="ucid" Type="string" Default="" />
      <Property Name="NomeUsuario" Type="string" Default="" />
      <Event Name="acao" />
      <Script Name="AfterShow" When="AfterShow">
        (function () {
          // JavaScript executed after render
        })();
      </Script>
      <Script Name="MontaMenu" When="Methods">
        function MontaMenu() { /* ... */ }
      </Script>
    </Definition>]]></Source>
  </Part>

  <Properties>
    <Property><Name>Name</Name><Value>UCMenuUsuario</Value></Property>
    <Property><Name>IsDefault</Name><Value>False</Value></Property>
  </Properties>
</Object>
```

> **`gx_read` cannot access UC scripts** — they live in Part `8e9e4a7c` CDATA.
> To read: `gx_export → gx_read_xpz`.
> To edit: `gx_export → gx_patch_xpz → gx_import`.

Raw `<` and `>` are allowed inside CDATA. The only forbidden sequence is `]]>`.

### SDT (`type` = `447527b5-...`)

```xml
<Object name="SdtVenMeusProjetosEmNegociacao" type="447527b5-9210-4523-898b-5dccb17be60a"
        guid="55cf69aa-6994-4632-8b64-6a97a0a4835d" parent="VEN" ...>

  <Part type="5c2aa9da-8fc4-4b6b-ae02-8db4fa48976a">
    <Level Name="SdtVenMeusProjetosEmNegociacao">
      <LevelInfo guid="9b4d4b27-..." name="SdtVenMeusProjetosEmNegociacao"
                 type="a76e9340-bdb9-445d-8f81-cfd4ddd0b0f3" description="...">
        <Properties>
          <Property><Name>AttCollection</Name><Value>False</Value></Property>
          <Property><Name>idCollectionItemName</Name><Value>Item</Value></Property>
        </Properties>
      </LevelInfo>

      <Item guid="..." name="UndId" type="f76e9340-bdb9-445d-8f81-cfd4ddd0b0f3" description="...">
        <Properties>
          <Property><Name>Name</Name><Value>UndId</Value></Property>
          <Property><Name>ATTCUSTOMTYPE</Name><Value>bas:Numeric</Value></Property>
          <Property><Name>Length</Name><Value>12</Value></Property>
          <Property><Name>Decimals</Name><Value>0</Value></Property>
        </Properties>
      </Item>

      <Item guid="..." name="ProjetoDisplay" ...>
        <Properties>
          <Property><Name>ATTCUSTOMTYPE</Name><Value>bas:VarChar</Value></Property>
          <Property><Name>Length</Name><Value>80</Value></Property>
        </Properties>
      </Item>
    </Level>
  </Part>

  <Properties>
    <Property><Name>Name</Name><Value>SdtVenMeusProjetosEmNegociacao</Value></Property>
    <Property><Name>IsDefault</Name><Value>False</Value></Property>
  </Properties>
</Object>
```

> **CRITICAL:** use `ATTCUSTOMTYPE` (`bas:Numeric`, `bas:VarChar`, `bas:Date`, `bas:DateTime`, `bas:Boolean`)
> for Item fields, **never** `idBasedOn=Attribute:ILJ_xxx`. The Attribute form makes the SDK use the
> source attribute's name as the field name, silently breaking the SDT structure.

### Procedure (`type` = `84a12160-...`)

```xml
<Object name="PrcVenRetXxx" type="84a12160-f59b-4ad7-a683-ea4481ac23e9" guid="..." ...>

  <!-- Source body — CDATA is MANDATORY; missing CDATA creates a null-blob ghost in the KB -->
  <Part type="e4c4ade7-53f0-4a56-bdfd-843735b66f47">
    <Source><![CDATA[
      Parm(in:&pGuid, in:&pSdtParm)
      // GeneXus code
      &NotificationInfo.Id      = &pGuid
      &NotificationInfo.Object  = !'WbcVenXxx'
      &NotificationInfo.Message = &SdtResult.ToJson()
      &WebNotification.NotifyClient(&WebNotification.ClientId, &NotificationInfo)
    ]]></Source>
  </Part>

  <!-- Rules (may be empty but Part must be present) -->
  <Part type="9b0a32a3-de6d-4be1-a4dd-1b85d3741534">
    <Source><![CDATA[ ]]></Source>
  </Part>

  <!-- Variables -->
  <Part type="e4c4ade7-53f0-4a56-bdfd-843735b66f47">
    <Variable Name="pGuid">
      <Properties>
        <Property><Name>ATTCUSTOMTYPE</Name><Value>bas:Character</Value></Property>
        <Property><Name>Length</Name><Value>36</Value></Property>
      </Properties>
    </Variable>
    <Variable Name="NotificationInfo">
      <Properties>
        <Property><Name>idBasedOn</Name><Value>sdt:NotificationInfo, GeneXus.Server</Value></Property>
      </Properties>
    </Variable>
    <Variable Name="SdtResult">
      <Properties>
        <Property><Name>idBasedOn</Name><Value>sdt:SdtVenXxx</Value></Property>
      </Properties>
    </Variable>
    <Variable Name="CollectionResult">
      <Properties>
        <Property><Name>idBasedOn</Name><Value>sdt:SdtVenXxx</Value></Property>
        <Property><Name>AttCollection</Name><Value>True</Value></Property>
      </Properties>
    </Variable>
  </Part>
</Object>
```

### WebPanel / WebComponent (`type` = `c9584656-...`)

```xml
<Object name="WbcVenXxx" type="c9584656-94b6-4ccd-890f-332d11fc2c25" guid="..." ...>

  <!-- Events -->
  <Part type="c44bd5ff-f918-415b-98e6-aca44fed84fa">
    <Source><![CDATA[
      Event Start
        If &HttpRequest.Method = 'GET'
          &Guid = GUID.NewGuid().ToString()
        EndIf
      EndEvent
      Event Refresh
        If &HttpRequest.Method <> 'GET'
          Composite
            SectionLoading.Visible = True
            PrcVenRetXxxSubmit.Submit(!"", &Guid, &SdtContext, &SdtParm)
          EndComposite
        EndIf
      EndEvent
      Event OnMessage(&NotificationInfo)
        If &NotificationInfo.Id = &Guid
          &Collection.FromJson(&NotificationInfo.Message)
          // bind grid
          SectionLoading.Visible = False
        EndIf
      EndEvent
    ]]></Source>
  </Part>

  <!-- WebForm (GxMultiForm) -->
  <Part type="d24a58ad-57ba-41b7-9e6e-eaca3543c778">
    <Source><![CDATA[<GxMultiForm>
      <!-- layout XML -->
    </GxMultiForm>]]></Source>
  </Part>

  <!-- Variables (same structure as Procedure) -->
  <Part type="e4c4ade7-53f0-4a56-bdfd-843735b66f47">
    <!-- ... -->
  </Part>

  <Properties>
    <Property><Name>Name</Name><Value>WbcVenXxx</Value></Property>
    <Property><Name>Theme</Name><Value>c804fdbd-7c0b-440d-8527-4316c92649a6-3</Value></Property>
    <Property><Name>WEB_COMP</Name><Value>Yes</Value></Property>
    <Property><Name>WebUX</Name><Value>SMOOTH</Value></Property>
  </Properties>
</Object>
```

> **`Theme` and `WebUX` are mandatory properties for WBC objects.** Missing them causes
> `NullReferenceException` in `gx_modify` even for unrelated changes. Fix via SQL UPDATE on
> `EntityVersion.EntityVersionProperties`.

## 7. Variable Typing in XPZ

| Type | XPZ Syntax |
|---|---|
| KB attribute | `idBasedOn=Attribute:ILJ_xxx` ⚠️ breaks cross-KB |
| Domain | `idBasedOn=Domain:NomeDominio` ✅ safe |
| Project SDT | `idBasedOn=sdt:SdtVenXxx` |
| External SDT | `idBasedOn=sdt:NotificationInfo, GeneXus.Server` |
| Character | `ATTCUSTOMTYPE=bas:Character` + `Length` |
| VarChar | `ATTCUSTOMTYPE=bas:VarChar` + `Length` |
| Numeric | `ATTCUSTOMTYPE=bas:Numeric` + `Length` + `Decimals` |
| Date | `ATTCUSTOMTYPE=bas:Date` |
| DateTime | `ATTCUSTOMTYPE=bas:DateTime` |
| Boolean | `ATTCUSTOMTYPE=bas:Boolean` |
| Collection | `AttCollection=True` on the **variable** (NOT on `LevelInfo` of the SDT) |

> `AttCollection=True` on `LevelInfo` makes every variable of that SDT type a collection → `src0216: invalid property` when accessing `.Field` directly.

## 8. Silent Import Failure Modes

GeneXus imports without error but without effect when any of the following is true:

1. `lastUpdate` in XPZ ≤ the existing version in the target KB
2. A variable references an SDT that does not exist in the target KB
3. A Control or Section referenced in events does not exist in the WebForm
4. A called procedure has an incompatible signature
5. Object GUID collides with a **different** object already in the target KB

**Diagnosis:** query `EntityVersion` count before and after — `delta = 0` means silent failure:

```sql
SELECT COUNT(*) FROM EntityVersion WHERE EntityTypeId = 34 AND EntityId = <id>
```

**Fix:** bump `lastUpdate` to a future timestamp + zero `checksum` (`"00000000000000000000000000000000"`) + resolve all prerequisites in the target KB before importing the dependent object.

## 9. Reading an XPZ in PowerShell

```powershell
Add-Type -AssemblyName System.IO.Compression.FileSystem

$zip = [System.IO.Compression.ZipFile]::OpenRead("C:\KBs\FoccoLojas_02\FoccoLojas_03.xpz")
$reader = [System.IO.StreamReader]::new($zip.Entries[0].Open())
$xmlText = $reader.ReadToEnd()
$reader.Close()
$zip.Dispose()

$xml = [xml]$xmlText

# List all objects
$xml.ExportFile.Objects.Object | Select-Object name, type, description | Format-Table

# Get a specific object
$obj = $xml.ExportFile.Objects.Object | Where-Object { $_.name -eq "SdtVenMeusProjetosEmNegociacao" }

# Get a specific Part by GUID
$sdtLevelPartGuid = "5c2aa9da-8fc4-4b6b-ae02-8db4fa48976a"
$part = $obj.Part | Where-Object { $_.type -eq $sdtLevelPartGuid }
```

## 10. Generating a Minimal XPZ for Import

```powershell
Add-Type -AssemblyName System.IO.Compression.FileSystem

$xmlContent = @'
<?xml version="1.0" encoding="utf-8"?>
<ExportFile>
  <KMW><MajorVersion>4</MajorVersion><MinorVersion>0</MinorVersion><Build>177934</Build></KMW>
  <Source kb="9ced92e5-f7b4-47c3-9af8-c84b8f5c565a"
          username="COMPUSOFT\lucas.karsten"
          UNCPath="\\host\C$\KBs\FoccoLojas_03">
    <Version guid="64e77778-2ac3-4a8b-b0c3-e641b98d1943" name="FoccoLojas_03" />
  </Source>
  <Objects>
    <!-- ONLY the target objects here — do not include unrelated hosts -->
  </Objects>
</ExportFile>
'@

$tmpDir  = "$env:TEMP\xpz_build"
$tmpXml  = "$tmpDir\export.xml"
$outXpz  = ".\output\MeuObjeto.xpz"

New-Item -ItemType Directory -Force $tmpDir | Out-Null
[System.IO.File]::WriteAllText($tmpXml, $xmlContent, [System.Text.Encoding]::UTF8)
if (Test-Path $outXpz) { Remove-Item $outXpz }
[System.IO.Compression.ZipFile]::CreateFromDirectory($tmpDir, $outXpz)
```

> Alternatively use Python: `zipfile.ZipFile(xpz, 'w').write(xml, arcname='export.xml')`
